What is Akka?

  • open source
  • actors
  • event-driven
  • JVM

Why Akka?

  • scale up (concurrency)
  • scale out (distributed)
  • fault tolerant

Akka Basics

  • message queue
  • dispatcher gives actor a thread and a message
  • lightweight (make millions of actors)
  • actors share no state

Why Akka + Scala?

  • case classes
  • pattern matching

Scala case classes

  • immutable state
  • use as actor messages

Scala pattern matching

  • elegant conditional logic
  • use to decide how to handle messages
